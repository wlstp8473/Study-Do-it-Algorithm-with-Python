# 동일성 판단
동일성이란 내부의 값 및 식별번호까지 모두 동일한 객체를 의미 (is를 통한 동일성 판단)
ex) list_02 = list_01
    내부적으로 같은 객체를 참조하기 때문에, list_02의 원소값이 변경되면 list_01의 원소값에도 영향을 미친다.

#등가성 판단
단순히 내부의 값만 같은지를 비교
ex) list01 = [1, 2, 3, 4, 5]와 list_02 = [1, 2, 3, 4, 5]가 있을 때
    list_01 == list_02 의 결과는 True
    list_01 is list_02 의 결과는 False


# 함수의 어노테이션(주석)
파이썬 문법은 제약성이 적어 유연성이 높지만, 그로 인한 단점도 존재한다.
자료형 선언 없이 변수나 함수를 자유롭게 사용할 수 있지만 명시적으로 해석하기 어려운 경우가 존재한다.
그로 인해 annotation(주석 달기) 기능을 제공한다.
어노테이션은 강제성은 없지만, 함수의 매겨변수나 반환값 등을 알려주는 등... 코드를 이해하는데 도움을 준다.
주석의 내용은 코드 실행에 영향을 주지 않는다.
